2. Microphone — Use Your Computer's Built-in Mic
Yes — your computer's built-in microphone works perfectly for this project. Vosk and sounddevice access any system audio device by index. You're sitting close to a fixed robot arm, so noise rejection matters less than for a streaming setup.

Why it works
	•	Vosk is trained on noisy real-world speech — it handles fan noise and room echo well at short range.
	•	sounddevice defaults to the system's default input device. No configuration needed.
	•	The Sound Sensor Module in your kit is NOT usable for this — it only outputs a HIGH/LOW signal when sound exceeds a threshold, not actual audio waveforms.

One-line code to list your mic devices
import sounddevice as sd
print(sd.query_devices())  # find your mic's device index
# Then in voice.py: sd.InputStream(device=INDEX, ...)

If voice recognition is unreliable with the built-in mic, the FIFINE K669B is still the recommended upgrade at ~$25 on Amazon — but try the built-in first.

Integration code — swap between all three with one variable
# config.py — change this one line to switch providers
LLM_PROVIDER = "groq"  # or "google" or "ollama"

# ai/llm_agent.py
from openai import OpenAI

CONFIGS = {
    "groq":   {"api_key": "gsk_YOUR_KEY",  "base_url": "https://api.groq.com/openai/v1",  "model": "llama-3.3-70b-versatile"},
    "google": {"api_key": "AI_YOUR_KEY",   "base_url": "https://generativelanguage.googleapis.com/v1beta/openai/", "model": "gemini-2.5-flash"},
    "ollama": {"api_key": "ollama",         "base_url": "http://localhost:11434/v1",        "model": "llama3.2:3b"},
}

cfg = CONFIGS[LLM_PROVIDER]
client = OpenAI(api_key=cfg["api_key"], base_url=cfg["base_url"])

def ask_llm(prompt):
    resp = client.chat.completions.create(model=cfg["model"], messages=[{"role":"user","content":prompt}])
    return resp.choices[0].message.content

6. Updated System Architecture
One significant bonus: your Arduino Mega replaces the UNO. This matters because the Mega has four independent hardware UART ports — you can connect the TF-Luna LiDAR on Serial1, the Python host on Serial0 (USB), and reserve Serial2/3 for future sensors, all simultaneously with no conflicts.

Layer
Component
Input
Voice: computer built-in mic → Vosk STT → asyncio queue. Gesture: webcam → MediaPipe (existing). Vision: webcam YOLO + TF-Luna LiDAR on Mega Serial1.
Orchestrator
Python asyncio event loop on host computer. Routes events to processing modules.
AI / LLM
Groq free tier (Llama 3.3 70B) or Ollama local — configured via single variable in config.py.
Motion
IK solver (ikpy) + trajectory smoother → serial packet → Mega Serial0 (USB).
Hardware
Arduino Mega: ULN2003 (lift stepper on D8–D11), SG90 servos (D3, D6, D9), TF-Luna (Serial1 at 115200). Feedback over Serial0.

The only thing you need to purchase is a LiDAR sensor for Phase 3 vision. The TF-Luna at ~$22 USD is the recommended pick. Before even buying that, you can prototype the pick-and-place logic using the HC-SR04 ultrasonic sensor already in your kit.
